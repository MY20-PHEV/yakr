What is Yakr?

Yakr is a decentralised messaging protocol. Alice encrypts a message, stores an opaque blob at a friend's relay, and Bob fetches it later — even while offline. Relays never see plaintext. Contacts are established pairwise via signed invites, not global user IDs.

  R -->|opaque blob| R
  B[Bob Client] -->|fetch + decrypt| R

Core claim: Offline delivery works without a central server. The relay only validates structure, size, expiry, and optional auth tickets — never message content.

Quick Example

Alice sends "hello vector" to Bob while Bob is offline:

1. Build inner message

JSON payload with conversation ID, sequence number, and body text.

2. Encrypt with session key

message_key(seq=1) from pairwise master secret → XChaCha20-Poly1305.

3. Derive mailbox tag

HMAC over alice->bob|{epoch} — relay sees only the 32-byte tag.

4. POST to relay

POST /v1/blobs with tag, expiry, and ciphertext. Relay stores opaque data.

5. Bob fetches later

GET /v1/blobs/{tag} for current + lookback epochs, decrypt, display message.
