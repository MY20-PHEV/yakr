Overview & Actors

Who participates in Yakr, what they trust, and how the reference implementation is organised across nine phases.

Trust Model

flowchart LR
  subgraph trusted [Client trusts]
    T1[Own device keys]
    T2[Contact's signed invites/profiles]
    T3[Safety code verified OOB]
  end
  subgraph untrusted [Relay assumed malicious]
    U1[Cannot decrypt blobs]
    U2[Can drop/delay/limit]
    U3[Sees tags + sizes + timing]
  end
Relays are dumb by design. They validate tag length (32 B), future expiry, size ≤ 64 KiB, per-tag blob caps, and optional relay tickets. They never parse chat semantics.
